# Mag88（Responsive Magazine Site）

CodeJump初級編の第2課題。マガジン形式のレイアウトをレスポンシブで再現実装しました。  
「同じコードなのになぜ結果が違うのか」という技術的な壁を、**CSS詳細度（Specificity）の理解**によって自力で突破した記念碑的なプロジェクトです。

---

## 使用技術
- **HTML5** / **CSS3**
- **Flexbox**（多段レイアウト、位置制御）
- **メディアクエリ**（`max-width: 896px`）
- **ress.min.css**（リセットCSS）

---

## 実装内容
- **マガジン記事セクション**: `position: absolute` を用いた画像上へのテキスト配置と中央寄せ
- **Catalog & Antique**: `flex-direction: column` によるスマホ最適化と余白調整
- **フッター構造**: 3カラムから1カラムへの切り替えと、ロゴのセンター配置
- **詳細度の制御**: PC版とスマホ版のスタイル干渉を解決するセレクタ設計

---

## 学んだこと（今回の核心）
- **CSS詳細度の罠**: PC版で記述した `#footer .flex-item .item` という強いセレクタが、スマホ版の標準的な指定を上書きしてしまう問題を特定。セレクタの強さを合わせる重要性を学んだ。
- **リセットの技術**: PC版で設定した `padding: 60px 0;` などの余白を、スマホ版（メディアクエリ内）で適切にリセット（`0`）する手順を習得。
- **検証ツールの活用**: ブラウザのデベロッパーツールを使い、どのスタイルが優先されているかを視覚的に分析する能力が向上した。

「手本通りに打つ」だけでなく、ブラウザがコードをどう解釈し、どのルールを優先しているかという「CSSの優先順位」を深く理解することができた。中指一本のタイピングで一文字ずつ積み上げた結果、自力で原因を特定し、完遂できたことは大きな自信になった。
